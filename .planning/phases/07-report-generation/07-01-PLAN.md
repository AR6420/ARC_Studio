       - color_code_score(metric_name: str, value: float) -> str:
         For metrics in INVERTED_SCORES (backlash_risk, polarization_index): <30 green, <60 amber, >=60 red.
         For other metrics: >=70 green, >=40 amber, <40 red.
         Import INVERTED_SCORES from optimization_loop.

       - class ReportGenerator:
         Constructor: __init__(self, claude_client: ClaudeClient) -- follows ResultAnalyzer pattern.

         async def generate_report(self, campaign: CampaignResponse, all_iterations: list[IterationRecord], all_analyses: list[AnalysisRecord], best_scores_history: list[dict], stop_reason: str) -> dict[str, Any]:
           Orchestrates all 4 layers. Returns dict with keys: verdict, scorecard, deep_analysis, mass_psychology_general, mass_psychology_technical.

         Private methods:
         a) async def _generate_verdict(self, campaign, final_variants_with_scores, thresholds_met, iterations_run) -> str:
            - Use find_best_composite() on the final iteration's composite scores to identify the winner.
            - Build winning_variant dict from the winning IterationRecord (variant_id, content, strategy, composite_scores as dict, tribe_scores as dict, mirofish_metrics as dict).
            - Build all_variants list similarly from all final iteration records.
            - Call self._claude.call_opus(system=REPORT_VERDICT_SYSTEM, user=build_report_verdict_prompt(...), max_tokens=2048).
            - Return the text.

         b) def _assemble_scorecard(self, campaign, all_iterations, best_scores_history, stop_reason, thresholds_met, scorecard_rationale) -> dict:
            - PROGRAMMATIC -- no LLM call for the data.
            - Get final iteration records. Rank variants using find_best_composite logic: compute adjusted average for each variant, sort descending.
            - For each variant, compute color_coding dict using color_code_score() for each composite score.
            - Build iteration_trajectory from best_scores_history (iteration number + scores dict).
            - Build thresholds_status from check_thresholds() if campaign.thresholds exists.
            - Return ScorecardData-compatible dict with winning_variant_id, variants list (ranked), iteration_trajectory, thresholds_status, and summary from scorecard_rationale.

         c) def _assemble_deep_analysis(self, all_iterations, all_analyses) -> dict:
            - Pure data aggregation. NO LLM call (per research -- Layer 3 is raw data).
            - Group iterations by iteration_number.
            - For each iteration_number, build: {"iteration": N, "variants": [{variant_id, content, strategy, tribe_scores (as dict or None), mirofish_metrics (as dict or None), composite_scores (as dict or None)}], "analysis": analysis_json from matching AnalysisRecord}.
            - Return {"iterations": [...]}.

         d) async def _generate_psychology(self, campaign, final_variants_with_scores, winning_variant) -> tuple[str, str]:
            - Resolve demographic description (get_profile if not custom).
            - Build winning_variant dict and all_variants list from final iteration records.
            - Build simulation_summary from winning variant's mirofish_metrics (as dict).
            - Call self._claude.call_opus for GENERAL (system=MASS_PSYCHOLOGY_GENERAL_SYSTEM, user=build_mass_psychology_general_prompt(...), max_tokens=2048). Returns general text.
            - Call self._claude.call_opus for TECHNICAL (system=MASS_PSYCHOLOGY_TECHNICAL_SYSTEM, user=build_mass_psychology_technical_prompt(..., tribe_scores=winning_tribe_scores), max_tokens=2048). Returns technical text.
            - Return (general_text, technical_text).
            NOTE: Per Open Question #1 in RESEARCH.md, we keep these as 2 separate calls to ensure distinct system prompts produce quality output. The combined approach risks degrading the technical section. Total Opus calls for report: 3 (verdict + general + technical). Plus scorecard rationale if needed. Budget: within acceptable range.
