    **ui/src/hooks/use-progress.ts:**
    Custom hook following RESEARCH.md Pattern 2 exactly. This is critical to get right.

    ```typescript
    export function useProgress(campaignId: string | null)
    ```

    Returns: { events: ProgressEvent[], latestEvent: ProgressEvent | null, isConnected: boolean, isComplete: boolean, currentStep: string | null, progress: { iteration: number, maxIterations: number, stepIndex: number, totalSteps: number, etaSeconds: number | null } }

    Implementation details:
    - Store EventSource ref in useRef to manage lifecycle
    - State: events array, isConnected, isComplete
    - On connect (campaignId is truthy and not isComplete):
      - Create EventSource to `${API_BASE}/api/campaigns/${campaignId}/progress`
      - Set onopen -> setIsConnected(true)
      - Set onerror -> setIsConnected(false), close EventSource
      - Register addEventListener for ALL named event types:
        'iteration_start', 'step_start', 'step_complete', 'iteration_complete',
        'threshold_check', 'convergence_check', 'campaign_complete', 'campaign_error'
      - Each listener: parse JSON from e.data, append to events, check for terminal event
      - Per Pitfall 2: DO NOT use es.onmessage (won't receive named events)
    - Terminal events: campaign_complete, campaign_error -> setIsComplete(true), close EventSource
    - Per Pitfall 4: On campaign_complete, call queryClient.invalidateQueries({ queryKey: ['campaign', campaignId] })
      - Import useQueryClient from @tanstack/react-query to get queryClient reference
    - Cleanup: useEffect returns function that closes EventSource (per Pitfall 6)
    - Only connect when campaignId is provided and not already complete
    - Compute derived state from latestEvent: currentStep, progress summary
