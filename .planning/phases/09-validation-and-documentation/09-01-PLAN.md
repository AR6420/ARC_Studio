  <action>
Create two Python scripts per D-02 (automated run script + validation checker):

**scripts/run_validation.py** — Automated scenario runner:
- Reads all JSON briefs from `scenarios/` directory
- For each scenario, invokes `python -m orchestrator.cli` via `subprocess.run()` with:
  - `--seed-content` from brief's `seed_content` field
  - `--prediction-question` from brief's `prediction_question` field
  - `--demographic` from brief's `demographic` field
  - `--agent-count` from brief's `agent_count` field
  - `--max-iterations` from brief's `max_iterations` field
  - `--output` set to `results/{scenario_name}_result.json` (auto-creates results/ dir)
  - `--verbose` flag
- Creates `results/` directory if it does not exist
- Prints progress: "Running scenario N/5: {name}..."
- Captures return code; if non-zero, logs error and continues to next scenario
- After all scenarios, prints summary: "X/5 scenarios completed successfully"
- Accepts optional `--scenario` argument to run a single scenario by name
- Accepts optional `--dry-run` flag that prints the CLI commands without executing

**scripts/validate_results.py** — Results quality checker:
- Reads all result JSON files from `results/` directory
- For each result, checks three criteria:

1. **Iteration improvement (VAL-03):** Compare composite scores between iterations. For each scenario, check if the best variant's average composite score in the last iteration is higher than the first iteration. Count how many scenarios show improvement. Report pass/fail against 4/5 threshold per D-03.

2. **Cross-system reasoning (VAL-04):** Search the `analysis.cross_system_insights` array in each iteration for strings containing BOTH "TRIBE" (or "neural" or "brain" or "attention" or "emotional" or "memory") AND "MiroFish" (or "simulation" or "social" or "share" or "propagation"). Report pass/fail per scenario — all 5 must pass.

3. **Demographic sensitivity (VAL-05):** Compare results across scenarios that use different demographics. Check that composite score profiles differ meaningfully between scenarios (standard deviation of average scores across scenarios > 5 points). This is a heuristic check.

- Prints a structured report:
  ```
  VALIDATION REPORT
  =================
  Iteration Improvement: X/5 scenarios improved (need 4/5) [PASS/FAIL]
  Cross-System Reasoning: X/5 reports reference both systems (need 5/5) [PASS/FAIL]
  Demographic Sensitivity: Score variance = X.X (need > 5.0) [PASS/FAIL]

  Per-scenario details:
  ...
  ```
- Returns exit code 0 if all criteria pass, 1 otherwise
