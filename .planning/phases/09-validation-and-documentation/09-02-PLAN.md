Structure the README with these sections:

**1. Header and Overview**
- Project name: Nexus Sim
- One-paragraph description of the platform (content optimization via neural scoring + social simulation + LLM iteration)
- Core value proposition: iterative feedback loop produces measurably better content
- Status: Phase 1 POC

**2. Architecture Overview**
- ASCII diagram showing the 3-system feedback loop:
  ```
  User Input -> Claude Opus (variant generation)
                    |
            +-------+-------+
            |               |
      TRIBE v2          MiroFish
    (neural scores)   (social sim)
            |               |
            +-------+-------+
                    |
            Claude Opus (analysis)
                    |
              Iterate or Stop
  ```
- Brief explanation of each system:
  - TRIBE v2: Neural response prediction using LLaMA 3.2-3B, produces 7 brain-region scores
  - MiroFish-Offline: Multi-agent social simulation using Claude Haiku, produces 8 social metrics
  - Claude Opus: Cross-system analysis bridging neural and social data
- 7 composite scores explained (from Results.md Section 3.2)
- Iterative optimization: how each iteration improves on the last

**3. Prerequisites**
- Hardware: NVIDIA GPU with CUDA support (developed on RTX 5070 Ti)
- Software: Python 3.11+, Node.js 18+, Docker Desktop with NVIDIA Container Toolkit
- API keys: Anthropic API key, HuggingFace token (for LLaMA 3.2-3B gated model)

**4. Setup Instructions** (step-by-step)
- Clone the repo (including submodules for MiroFish)
- Copy .env.example to .env and fill in API keys
- Start infrastructure: `docker compose up -d` (Neo4j, Ollama, LiteLLM)
- Install Python dependencies: `pip install -e .` in orchestrator/
- Start TRIBE v2 scorer: `cd tribe_scorer && uvicorn main:app --port 8001`
- Start orchestrator: `cd orchestrator && uvicorn main:app --port 8000 --reload`
- Start UI: `cd ui && npm install && npm run dev`
- Verify: health check at http://localhost:8000/api/health

**5. Quick Start** (run your first campaign)
- Via UI: Navigate to http://localhost:5173, click New Campaign, fill form, Run
- Via CLI: `python -m orchestrator.cli --seed-content "..." --prediction-question "..." --demographic tech_professionals`
- Example with all options including --output for JSON export

**6. API Reference** (key endpoints)
- POST /api/campaigns — Create campaign
- GET /api/campaigns — List campaigns
- GET /api/campaigns/{id} — Get campaign detail
- POST /api/campaigns/{id}/run — Start campaign execution
- GET /api/campaigns/{id}/progress — SSE progress stream
- GET /api/campaigns/{id}/report — Get report
- GET /api/campaigns/{id}/report/export/json — JSON export
- GET /api/campaigns/{id}/report/export/markdown — Markdown export
- POST /api/estimate — Time estimate
- GET /api/health — System health check
- GET /api/demographics — List demographic presets
- For each endpoint: method, path, brief description, request/response shape

**7. Project Structure**
- Directory tree with brief description of each module
- Key files highlighted

**8. Configuration**

**9. Demographic Presets**

**10. Running Tests**
- `cd orchestrator && python -m pytest` for backend
- `cd ui && npm test` for frontend (if applicable)

**11. Validation**
